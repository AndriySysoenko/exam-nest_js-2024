## Description

Examination work on the backend development module, completed using the Nest.js framework in the TypeScript programming language.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# running docker server
$ npm run start:docker:db

# development
$ npm run start

# watch mode
$ npm run start:dev


```

## Run migrations

```bash
# creating migration
$ npm run migrate:generate

# application of migration
$ npm run migrate:run

# откат миграции
$ npm run migrate:reverte
```

## Deployment

When you are ready to deploy your NestJS application to production, there are a few key steps you can take to ensure it runs as efficiently as possible.
The project is built on a Docker container build. All the settings are already done and to get the container up and running you need to have Docker installed. Building the container provides a connection to the Postgres database and the Redis data store. The data for the server application to run properly is in the .env.example file. To work with it, you need to create a .env file in the root of the project and copy the contents of the .env.example file there.
